getVersions

  • getVersions(request: CatalogVersionRequest, abortSignal?: AbortSignal): Promise<VersionInfos>
    • Defined in olp-sdk-dataservice-read/lib/client/CatalogClient.ts:119

    Get the information about specific catalog versions. Maximum number of versions to be returned per call is 1000 versions. If range is bigger than 1000 versions 400 Bad Request will be returned.

    Parameters

    • request: CatalogVersionRequest

      Is CatalogVersionRequest instance and has ((getStartVersion)) and ((getEndVersion)) methods. StartVersion - Catalog start version (exclusive). Default is -1. By convention -1 indicates the virtual initial version before the first publication which will have version 0. EndVersion - Catalog end version (inclusive). If not defined, then the latest catalog version will be fethced and used.

    • Optional abortSignal: AbortSignal

      A signal object that allows you to communicate with a request (such as a Fetch) and abort it if required via an AbortController object @see https://developer.mozilla.org/en-US/docs/Web/API/AbortSignal

    Returns Promise<VersionInfos>

    A promise of the http response that contains the payload with versions in requested range.
